## ¿Qué he aprendido hoy?
- No es lo mismo memorizar que generalizar. Al memorizar responde bien ante los ejemplos vistos, pero no ante los ejemplos nuevos. Al generalizar el modelo aprende un patrón útil. 
- Para medir generalización, se dividen los datos en dos partes: train set y test set. Datos reservados para entrenar el modelo y los del test están reservados para evaluar.
- Ahora hay dos pérdidas: train loss y test loss. EL test loss es el error en datos nuevos que el modelo no vio. Si el train loss es muy baja y el test loss alta, significa que el modelo no generaliza bien. 
- Overfitting: Cuando un modelo se ajusta demasiado a los datos de entrenamiento y funciona mal en datos nuevos. Ha aprendido detalles específicos del train, pero no un patrón general. 
- Underfitting: Cuando el modelo no ha aprendido suficiente. 
- Es bueno primero mezclar los datos antes de dividirlos. Porque si están ordenados de alguna manera, el dataset del train puede ser muy distinto al del del test y salir mal. 
- A partir de ahora preguntaremos:
¿baja la pérdida en train?
¿cómo se comporta en test?
¿está generalizando?
- Normalizar:
1. dividir train/test
2. calcular min/max/media/desviación solo con train
3. transformar train usando esos valores
4. transformar test usando esos mismos valores
5. entrenar con train
6. evaluar con test
- Un modelo generaliza bien cuando mantiene buen rendimiento en datos nuevos, no vistos durante el entrenamiento.
