## ¿Qué he aprendido hoy?
- El problema de la escala de las features es que la feature con números más grandes puede dominar los gradientes. 
- Por eso se suelen escalar o normalizar datos antes de entrenar. 
- Normalización no es lo mismo que estandarización.
- La normalización min-max convierte los valores a un rango, normalmente entre 0 y 1.
- La estandarización transforma los datos para que tengan: media ≈ 0 y desviación estándar ≈ 1
- La normalización es útil cuando quieres dejar los valores en un rango conocido
- La estandarización es muy usada en Machine Learning clásico y redes neuronales cuando las features tienen distribuciones diferentes. 
- Escalar nunca cambia la información. Solo cambia la forma numérica. 
- Una vez que normalizas los pesos cambian de significado: ahora significan influencia sobre la feature normalizada.
- El escalado se aprende con el train y se aplica al test. No se aprende el escalado con el test. 
- Con la normalización se pierde la interpretación directa en unidades originales.
- Data leakage ocurre cuando información que debería estar reservada para evaluación se filtra en el entrenamiento.
